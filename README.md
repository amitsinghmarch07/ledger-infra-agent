# Ledger — Infra Agent Platform (prototype)

A working prototype of an AI agent that turns a plain-language project description
into a sized, priced infrastructure plan, then simulates a live monitoring console
with independent API-token and LLM-token budget tracking and data-driven optimization
recommendations. The React app is the UI layer; a FastAPI backend owns planning,
persistence, deployment state, monitoring simulation, recommendations, and export
generation.

This is a **simulation**: the "live" metrics are generated by the backend (random
walk + compounding growth), not pulled from a real cloud account or LLM provider.
It's meant as a functional demo of the product flow, not a production monitoring tool.

## What it does

1. **Describe** — type a requirement in plain language (e.g. "RAG pipeline, 10,000
   daily users, sub-2s latency"). The React UI sends it to the Python backend,
   which extracts user count, latency target, and workload type.
2. **Plan** — generates two infrastructure options (cost-optimized vs.
   latency-optimized) with compute sizing, model choice, and a full monthly cost
   breakdown.
3. **Console** — after you "deploy," the backend simulates live CPU/memory/GPU/latency/error
   metrics, and tracks API tokens and LLM tokens as separate budgets, each with a
   14-day usage history and a projected burn-rate chart (dashed line shows when
   you're on track to exceed budget).
4. **Optimize** — recommendations generated from the actual simulated numbers
   (prompt/completion token ratio, projected overrun dates, GPU idle time, error
   rate), not fixed copy.
5. **Export** — the backend generates a parameterized Terraform snippet and Dockerfile
   from whichever plan you deployed.

## Run it locally

Requires [Node.js](https://nodejs.org) 18+ and Python 3.11+.

```bash
npm install
python -m pip install -r backend/requirements.txt
python -m uvicorn backend.main:app --reload
npm run dev
```

Then open the URL Vite prints (usually `http://localhost:5173`). The Vite dev server
proxies `/api/*` requests to the FastAPI backend at `http://localhost:8000`.

## Build for production

```bash
npm run build
npm run preview
```

## Next steps if you want to make this real

- Swap the simulated backend metrics for real data: cloud provider cost/usage APIs
  (AWS Cost Explorer, GCP Billing), your LLM provider's usage API, and an
  observability backend (Prometheus/Datadog) for CPU/memory/latency/errors.
- Replace the heuristic backend parser with an actual LLM call for more robust
  intent extraction.
- Replace the local SQLite persistence with a production database.
- Wire the "Export" tab to actually provision infrastructure (e.g. run the
  generated Terraform via an API/CI job) instead of just displaying it.
